Introduction

Scalable Vector Graphics (SVG) is an XML-based markup language for describing two-dimensional graphics.

Essentially, a SVG is, for images, what a HTML is for text. It is a file extension that is written in XML syntax for describing images, as vector graphics, in an open Web standard. It can render any vector cleanly at any size and is designed specifically to work well with other web standards including CSS, DOM, JavaScript, and SMIL.

SVG has been developed by the World Wide Web Consortium (W3C) since 1999.

What is a vector?

When you are working with digital images, there are two types of image: raster and vector. The images built by vectors, are in fact being built by mathematical formulas that establish points on a grid. Making it possible to resize the image without losing quality.

Basic properties
  • The first property to keep in mind is that the order of organization of the elements, from background to foreground, is defined in the sequence of text lines in the file. Therefore, the further down the lines of code, the more ahead of the elements it will be. So normally your background should be stated in the first line. To better understand this concept in practice, take a look at the z-index CSS property
  • There are several different ways to use SVG images in CSS, HTML or direclty in the browser:
    • If the HTML is XHTML and is delivered as type application/xhtml+xml, the SVG can be directly embedded in the XML source.
    • The SVG can also be directly embedded in HTML, using this syntax: <embed src="image.svg"/>
    • It can be added with a <img> using this syntax: <img src="image.svg" alt="My SVG"/>
    • The SVG file can be referenced with an object element, with <object data="image.svg" type="image/svg+xml">
    • Likewise an iframe element can be used: <iframe src="image.svg">
    • An CSS property of background-image can be used as: body { background-image: url(image.svg); }
    • SVG images can be written directly into the HTML document using the <svg> tag: <body> <svg> write the code here </svg> </body>
    • Finally, SVG can be created dynamically with JavaScript and injected into the HTML DOM.
The Grid

For all elements, SVG uses a coordinate system or grid system. The top left corner of the document is considered to be the point (0,0), or point of origin. Positions are then measured in pixels from the top left corner, with the positive x direction being to the right, and the positive y direction being to the bottom.

Example

<rect x="0" y="0" width="100" height="100" />

Defines a rectangle from the upper left corner that spans 100px to the right and 100px to the bottom.

Units
In a SVG file there are several possibilities of different units.
  • Pixels (px): SVG files can use pixels as a unit of measurement, just like in raster images. For example, you can specify the dimensions of an SVG element or the coordinates of shapes using pixels.
  • Percentages (%): Percentages are often used in SVG files to create responsive designs. For instance, you can set the width or height of an SVG element as a percentage of its containing element.
  • Inches (in): Inches are a common unit of measurement for physical print dimensions. You can use inches in SVG files to specify sizes for printing purposes.
  • Centimeters (cm): Centimeters are another unit of measurement for physical dimensions and are often used for print layouts.
  • Millimeters (mm): Millimeters are a finer unit of measurement than centimeters and are also used for precise physical dimensions in print layouts.
  • Points (pt): Points are a unit of measurement commonly used in typography and print design. They are often used to set font sizes and other text-related properties.
  • Picas (pc): Picas are another unit used in typography and print design. One pica is equal to 12 points.
  • Em units (em): Em units are a relative unit of measurement often used for text sizing. One em is equal to the current font size. It can also be used for other properties, like spacing and positioning, in relation to the font size.
  • Ex units (ex): Ex units are another relative unit of measurement in SVG, typically used in relation to the height of lowercase letters in the current font.
  • User units (userUnit): User units are arbitrary units defined by the user or software. They are often used for custom scaling or transformations within the SVG file.

In the most basic case, one pixel in an SVG document maps to one pixel on the output device (i.e., the screen). But SVG wouldn't have the "Scalable" in its name if there weren't several possibilities to change this behavior. Much like absolute and relative font sizes in CSS, SVG defines absolute units (ones with a dimensional identifier like "pt" or "cm") and so-called user units, which lack that identifier and are plain numbers.

Basic Shapes

The code to generate the image:

<svg width="600" height="60" version="1.1" xmlns="http://www.w3.org/2000/svg"> <rect x="10" y="10" width="30" height="30" stroke="black" fill="transparent" stroke-width="5" /> <rect x="60" y="10" rx="10" ry="10" width="30" height="30" stroke="black" fill="transparent" stroke-width="5" /> <circle cx="150" cy="25" r="20" stroke="red" fill="transparent" stroke-width="5" /> <ellipse cx="210" cy="25" rx="20" ry="5" stroke="red" fill="transparent" stroke-width="5" /> <line x1="260" x2="300" y1="10" y2="50" stroke="orange" stroke-width="5" /> <polyline points="310 10 315 20 320 15 325 30 330 25 335 40 340 35 345 50 350 45" stroke="orange" fill="transparent" stroke-width="5" /> <polygon points="400 10 405 30 420 30 410 40 415 55 400 45 385 55 390 40 380 30 395 30" stroke="green" fill="transparent" stroke-width="5" /> <path d="M450, 35 Q470, 10 480, 35 T520, 35" fill="none" stroke="blue" stroke-width="5" /> </svg>
Rectangle

The <rect> creates a rectangle on the screen. The tag has six attributes that are responsible for the shape and position.

<rect x="30" y="30" rx="5" ry="5" width="40" height="60"/>
  • x

    The x position of the top left corner of the rectangle.

  • y

    The y position of the top left corner of the rectangle.

  • width

    The width of the rectangle.

  • height

    The height of the rectangle.

  • rx

    The x radius of the corners of the rectangle.

  • ry

    The y radius of the corners of the rectangle.

Circle

The <circle> creates a circle on the screen. The tag has three attributes that are responsible for the size and position.

<circle cx="30" cy="30" r="5"/>
  • r

    The radius of the circle.

  • cx

    The x position of the center of the circle.

  • cy

    The y position of the center of the circle.

Ellipse

The <ellipse> creates a circle with x and y radius control. The tag has four attributes that are responsible for the size and position.

<cellipse cx="30" cy="30" rx="10" ry="15"/>
  • rx

    The x radius of the ellipse.

  • ry

    The y radius of the ellipse.

  • cx

    The x position of the center of the ellipse.

  • cy

    The y position of the center of the ellipse.

Line

The <line> creates a line, by the position (x, y) of two points. The tag has four attributes that are responsible for the position from start to end.

<line x1="30" x2="60" y1="30" y2="60" stroke="rgb(0, 127, 255)" stroke-width="5"/>/>
  • x1

    The x position of point 1.

  • x2

    The x position of point 2.

  • y1

    The y position of point 1.

  • y2

    The y position of point 2.

Polyline

The <polyline> connect each point with a straight line, by the position (x, y) of n points. The tag has one attribute.

<polyline points="60 110 65 120 70 115 75 130 80 125 85 140 90 135 95 150 100 145"/>
  • points

    A list of points. Each number must be separated by a space, comma, EOL, or a line feed character with additional whitespace permitted. Each point must contain two numbers: an x coordinate and a y coordinate. So, the list (0,0), (1,1), and (2,2) could be written as 0, 0 1, 1 2, 2.

Polygon

The <polygon> connect each point with a straight line, by the position (x, y) of n points, similar to <polyline> though, the only difference being that the start and end points are automatically connected. The tag has one attribute.

<polygon points="50, 160 55, 180 70, 180 60, 190 65, 205 50, 195 35, 205 40, 190 30, 180 45, 180"/>
  • points

    A list of points, each number separated by a space, comma, EOL, or a line feed character with additional whitespace permitted. Each point must contain two numbers: an x coordinate and a y coordinate. So, the list (0,0), (1,1), and (2,2) could be written as 0, 0 1, 1 2, 2. The drawing then closes the path, so a final straight line would be drawn from (2,2) to (0,0).

Path

The <path> is a generic tag that you can construct any kind of shape. The tag has one attribute.

<path d="M20,230 Q40,205 50,230 T90,230" fill="none" stroke="rgb(0, 127, 255)" stroke-width="5"/>
  • d

    A list of points and other information about how to draw the path.
